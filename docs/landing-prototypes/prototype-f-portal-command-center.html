What the Portal proves

AFK does not mean invisible.

The product promise only works if developers can see what happened. Sandman records durable state and exposes it through CLI commands and the Portal, so sleeping through the run does not mean losing control of the work.

Status without chat scrollback

Runs move through queued, running, reviewing, completed, failed, and archived states.

Blockers stay explicit

Tickets created from `to-tickets` keep their blocking edges. Sandman can run the ready frontier instead of guessing order.

Logs stay attached

When an agent fails or stalls, the run leaves logs, branches, task files, and events behind.

Review modes

Use Sandman's review daemon, or bring your own reviewer.

Sandman can run `sandman review` as a daemon that responds to review requests. Or you can use OpenCode, Codex, Copilot, CodeRabbit, or any review agent that attends to comments on the PR. The important part is not the reviewer brand. The important part is that PR review is a gate, not an afterthought.

Review optionHow it fitsWhy it matters
`sandman review` daemonListens for review requests and runs the configured review workflow.Integrated with Sandman's AFK operating model.
OpenCode or Codex reviewerResponds to PR comments such as a configured review trigger.Lets teams use their preferred review agent.
Human reviewerReviews the PR like normal, with logs and spec links available.Preserves the same accountability as traditional development.
