# ADR-0030: Standardize per-row RunID and run directory naming

## Status

accepted

## Context

Sandman batches runs into a shared `<batchDir>` named after a batch-scoped `<ts>-<sid>` prefix generated by `runid.NewBatch()`. Each individual run inside a batch needs a stable, unique, human-readable identifier that also serves as its on-disk folder name under `<batchDir>/runs/<runID>/`.

Prior to this decision, review runs used a literal `"review"` subject and a `runs/review/` alias, creating ambiguity with ordinary AgentRuns and making it impossible to run multiple independent review sessions concurrently against the same batch.

Issue #1551 introduced a canonical per-row RunID contract that makes review runs ordinary AgentRuns with a deterministic name derived from the PR number and optional linked issue.

## Decision

All per-row RunIDs share the shape `<ts>-<sid>-<subject>`, where `<ts>-<sid>` is the batch prefix and `<subject>` is computed per kind.

### Issue-driven runs

Issue-driven runs use `runid.NewBatchID(runid.KindIssue, n, firstSubject, ts, shortid)`:

- **Single issue (N=1):** `<ts>-<sid>-<firstSubject>`
- **Multiple issues (N>1):** `<ts>-<sid>-<firstSubject>+N-1`

### Review runs

Review runs use `runid.NewRunID(runid.KindReview, subject, ts, shortid)` composed by `internal/review.ReviewRunIDFor(prNumber, linkedIssue, ts, shortid)`:

- **Orphan review (no linked issue):** subject is `PR<pr>`, RunID is `<ts>-<sid>-PR<pr>`
- **Linked review (linked issue):** subject is `<linkedIssue>-PR<pr>`, RunID is `<ts>-<sid>-<linkedIssue>-PR<pr>`

The linked issue number is obtained from `(*github.PR).LinkedIssueNumber()`. Pass `0` when the PR does not link an issue.

The `<ts>-<sid>` prefix is owned by `runid.NewBatch()` and threaded through unchanged. The per-row subject portion is built by `ReviewRunIDFor`. The resulting `<ts>-<sid>-<linkedIssue?>-PR<pr>` shape is the canonical per-row RunID: every review run lands under a folder whose name is exactly its per-row RunID.

The legacy literal `"review"` alias and the `runs/review/` folder name are explicitly NOT used by any current code path.

### Prompt-only runs

Prompt-only runs (no associated issue) use `runid.NewBatchID(runid.KindPromptOnly, ...)` or `runid.NewRunID(runid.KindPromptOnly, ...)`:

- **No subject:** `<ts>-<sid>-prompt`
- **With subject:** `<ts>-<sid>-prompt-<subject>`

## Consequences

### Positive

- Per-row RunID is deterministic: given the same batch prefix, PR number, and linked issue, the RunID is always the same.
- The on-disk path `<batchDir>/runs/<runID>/` directly reflects the RunID, making runs navigable and log aggregation trivial.
- Multiple concurrent review sessions against the same batch are possible because each gets a unique per-row RunID.
- Human-readable: `20250720-abc123-PR99` immediately conveys that this is a review run for PR 99.

### Negative

- The `<ts>-<sid>-<subject>` shape is long for runs with many issues (`<ts>-<sid>-fix-login+47`).

### Neutral

- Review runs are ordinary AgentRuns; no special-case RunID kind is needed.
- The `runs/review/` legacy alias is gone but documented as superseded.
